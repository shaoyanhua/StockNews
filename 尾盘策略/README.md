# 尾盘隔夜策略 · 工具集

配套《[尾盘隔夜策略_视频原版与增强版.md](../尾盘隔夜策略_视频原版与增强版.md)》的机械化执行与验证工具。

## 文件

| 文件 | 作用 |
|---|---|
| `screener_v1.py` | 增强版V1筛选器。盘中 14:35–14:50 运行=实时筛选；收盘后运行=回放最近交易日复盘。输出候选表并落 `records/signals_日期.csv`（第七章模板） |
| `fill_results.py` | 次日 10:05 后运行，按 §5 退出规则机械回填 开盘/9:45/10:00/收益/超额 |
| `download_daily.py` | 夜间刷新日线缓存（候选池约3500只，qfq+raw各650根），供回测与筛选器使用 |
| `backtest_daily.py` | 日线降级回测（原版近似 + V1近似 + 裸动量对照 + 全市场基准） |
| `backtest_f1.py` | F1 改进策略回测（状态门/延长持有/跳空规则，均被证伪，见 [回测报告_F1策略.md](回测报告_F1策略.md)） |
| `data_sources.py` | 共用数据层：腾讯(行情/日K/分时) + 新浪(快照/行业) + 同花顺(题材)。已内置代理绕过与限流处理 |
| `records/` | 每日信号与回填记录（策略验证的原始台账） |
| `data/` | 日线缓存、行业映射缓存、回测输出 |

## 每日流程

```text
14:38  python screener_v1.py            # 初筛+确认（或 14:45 再跑一次收口）
15:30后 人工核对候选的 NA 项：公告/减持/解禁/龙头跳水（§3.5）
18:00  python download_daily.py         # 刷新日线缓存（增量，约几分钟）
次日
10:05  python fill_results.py           # 回填实际收益
```

Windows 任务计划（可选）：

```bat
schtasks /create /tn "尾盘筛选" /sc weekly /d MON,TUE,WED,THU,FRI /st 14:38 ^
  /tr "cmd /c cd /d h:\Project\StockNews\尾盘策略 && python screener_v1.py >> logs\screener.log 2>&1"
```

## 重要须知

1. **本机网络**：系统代理(127.0.0.1:1080)会拦国内行情源，`data_sources.py` 已自动绕过；
   东财、通达信TCP在本机不通，已弃用。腾讯K线接口有限流（4线程并发即触发501），
   下载器已限速+多域名轮换，**勿调高 WORKERS**。
2. **无信号是正常结果**（§7）：不为了交易降条件。筛选器输出"无信号"时照常落表。
3. **NA 项**：公告面(§3.5)、龙头跳水(§3.2.6)等无法机械化，候选表中列出，须人工确认后才算有效信号。
4. **回放模式的偏差**：收盘后回放用当日收盘值做粗筛（涨幅带放宽为1.0–8.5%再由分时精筛收口），
   与真实14:30快照有出入，仅用于复盘验证，不能替代盘中实跑。
5. **参数即版本**：`screener_v1.py` 顶部 `TH` 字典是文档默认阈值，改参数须复制新文件（V2），
   不得用新参数解释旧记录（§6.2）。
6. 回测结论与局限见 [回测报告_降级版.md](回测报告_降级版.md)。
